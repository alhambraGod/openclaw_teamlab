# 研究方向战略分析

适用场景：PI 需要从战略高度审视团队研究方向布局，发现新机会或评估差异化竞争力。
每周一 cron 自动触发（浅度版），或 PI 主动请求时执行深度版。

## 执行步骤

1. `curl -s "http://127.0.0.1:10301/api/agent/team-overview"` — 了解团队当前各成员的研究方向和阶段
2. `curl -s "http://127.0.0.1:10301/api/agent/meeting-details?recent_n=20"` — 从历次会议记录中挖掘隐性研究方向
3. `curl -s "http://127.0.0.1:10301/api/agent/team-analytics"` — 获取团队分析指标
4. `curl -s -X POST "http://127.0.0.1:10301/api/agent/query" -H "Content-Type: application/json" -d '{"sql":"SELECT ... FROM research_plans ..."}'` — 查询研究计划与方向
5. 综合以上数据，结合 memory 中的历史分析，生成战略报告

## 输出结构

```
## 研究方向战略分析 — [日期]

### 当前方向版图
（可视化各学生方向、聚类关系、覆盖的技术栈）

### 与全球前沿的差距
（团队方向 vs 近期 arXiv 热点：哪些在追、哪些在超、哪些缺失）

### 潜在新方向
（基于会议记录挖掘出的隐性方向，评估可行性）

### 全球竞争格局
（相似团队的规模、产出、资助情况，找差异化机会）

### 战略建议
（具体：哪个方向值得加强、哪个方向可以考虑收缩、有没有蓝海）
```

## 自动触发（cron 每周一 02:00）

- 执行浅度版（步骤 1-3）
- 生成摘要推送给 PI
- 完整分析写入 `memory/YYYY-MM-DD-direction.md`
- `curl -s -X POST "http://127.0.0.1:10301/api/agent/log-insight" ...` 保存关键发现

## 深度版（PI 主动请求）

- 执行全部 5 步
- 生成完整报告，可选发给 PI 的邮件或飞书消息
