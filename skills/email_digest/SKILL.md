# 技能：邮件摘要推送

## 技能名称
`email_digest` — 科研动态邮件摘要编写与发送

## 核心目标
将每日科研趋势监控结果、团队动态和重要通知整合为美观的 HTML 邮件，定时发送给团队成员。确保每位成员每天早上都能通过邮件快速了解领域最新进展和团队动态。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "发一份今天的科研动态邮件"
- "生成本周的周报邮件"
- "把最新的论文推送给大家"
- 定时触发：工作日早上8:00自动执行
- 包含关键词：邮件摘要、日报、周报、动态推送、邮件发送

## 输入格式
```json
{
  "digest_type": "string (可选，'daily'=日报, 'weekly'=周报, 默认'daily')",
  "recipients": ["string (可选，收件人邮箱列表，默认使用团队配置)"],
  "include_sections": ["string (可选，包含哪些内容板块)"],
  "custom_message": "string (可选，导师附加的自定义消息)",
  "send": "bool (可选，是否实际发送，默认true。false时仅预览)"
}
```

## 输出格式
```json
{
  "subject": "邮件主题",
  "html_body": "完整的HTML邮件内容",
  "recipients": ["收件人列表"],
  "sections_included": ["论文推荐", "团队动态", "行动项提醒"],
  "stats": {
    "papers_included": 10,
    "action_items_included": 5,
    "total_recipients": 12
  },
  "send_status": {
    "success": true,
    "sent_at": "2026-03-13 08:00:00",
    "failures": []
  }
}
```

## 执行步骤

### 第一步：收集内容素材
从各数据源收集邮件内容：
- `research_trend`：今日/本周的高相关论文
- `meeting_record`：近期会议的行动项进展
- `student_progress`：学生里程碑事件
- `direction_discovery`：新发现的研究方向（如有）
- 导师的自定义消息

### 第二步：编排邮件内容
调用 `compose_daily_digest(trends, team_info)` 生成 HTML 邮件：
- **标题区**：日期、问候语
- **论文推荐**：按相关度排序的论文卡片，含标题、摘要、链接
- **团队动态**：里程碑事件、新成果
- **行动项提醒**：临近截止或逾期的任务
- **导师寄语**：如有自定义消息
- **页脚**：退订链接、系统信息

### 第三步：邮件样式设计
HTML 邮件需兼容各邮件客户端：
- 使用内联CSS（不用外部样式表）
- 响应式设计（兼容手机阅读）
- 清晰的视觉层次和间距
- 团队品牌色和Logo

### 第四步：发送邮件
调用 `send_digest(recipients, subject, html_body)` 通过 SMTP 发送：
- 支持批量发送（BCC模式或逐一发送）
- 记录发送状态
- 处理发送失败（重试机制）

### 第五步：记录与反馈
- 保存邮件内容到数据库
- 记录发送状态和统计数据
- 追踪打开率和点击率（如支持）

## 注意事项

### 邮件质量
- 内容精炼，避免信息过载
- 每封邮件控制在5分钟内可读完
- 重要信息放在前面
- 论文推荐不超过10篇（日报）或20篇（周报）

### 技术要求
- HTML邮件需兼容 Outlook、Gmail、Apple Mail 等主流客户端
- 避免使用 JavaScript、外部CSS、复杂布局
- 图片使用绝对URL
- 提供纯文本备选版本

### 发送安全
- SMTP 凭据从环境变量读取，不硬编码
- 支持 TLS/SSL 加密连接
- 限制发送频率，避免被标记为垃圾邮件
- 包含退订链接

### 语言
- 邮件正文使用中文
- 论文标题保留英文原文
- 摘要翻译简洁准确

## 关联技能
- `research_trend`：提供论文推荐数据（必须先执行）
- `meeting_record`：提供行动项数据
- `student_progress`：提供学生里程碑数据
