# TeamLab PI 管理智能体

C-Si 科研团队专属 AI 助手，通过 **PI Agent**（后端 LLM Worker）统一处理所有团队管理任务。

## 使用方法（标准调用）

```bash
# 基础调用
$SKILLS_DIR/teamlab-pi-agent/scripts/teamlab.sh "任务描述"

# 飞书渠道（必须传入 sender_open_id，同步等待结果后回复）
$SKILLS_DIR/teamlab-pi-agent/scripts/teamlab.sh \
  "任务描述" \
  --user "feishu:{sender_open_id}" \
  --channel "feishu"

# 强制异步（不推荐：结果只能主动查询）
$SKILLS_DIR/teamlab-pi-agent/scripts/teamlab.sh \
  "任务描述" \
  --async
```

## 适用场景（一律使用此 skill）

✅ **使用此 skill：**
- "甄园昌老师跟谁合作最有价值？" → 合作分析
- "给我看看张旭华的学生档案" → 学生深度档案
- "我们团队的研究方向布局如何？" → 方向战略分析
- "生成本周团队周报" → 全团队周报
- "李明最近进展怎么样，有什么风险？" → 风险 + 成长叙事
- "为 AI+教育项目推荐最合适的合作团队" → 团队组建
- "查一下所有学生的行动项" → 行动项追踪

❌ **不适用（直接回答）：**
- 简单闲聊、与科研团队管理无关的问题

## 执行模式

| 渠道 | 模式 | 说明 |
|------|------|------|
| 飞书（`feishu:ou_xxx`） | **同步轮询**（180s内）| 等待结果直接回复飞书；超时告知任务 ID |
| Web UI / CLI | **同步轮询**（90s内）| 等待结果后输出；超时告知任务 ID |
| `--async` 参数 | **异步提交** | 仅返回任务 ID，需用户主动查询结果 |

## 重要规则

1. **人名必须先验证**：如果任务涉及具体姓名，脚本内部会自动处理验证，无需 Agent 额外调用验证接口
2. **固定 API 地址**：脚本内部自动解析后端地址，Agent 无需关心地址配置
3. **超时自动处理**：超过 3 分钟的任务会收到友好提示，不会阻塞 Agent
