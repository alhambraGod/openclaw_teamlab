# 技能：科研方向发现

## 技能名称
`direction_discovery` — 科研新方向发现与评估

## 核心目标
从团队会议记录、日常讨论中提取研究主题关键词，通过系统性地对主题进行排列组合，评估每种组合的科研潜力，结合全球研究趋势，发现值得探索的新科研方向。这是一个从"已知"推向"未知"的创造性过程。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "帮我想想有什么新的研究方向"
- "最近讨论的话题有没有可以交叉创新的"
- "把我们的研究主题排列组合一下看看"
- "分析一下哪些方向组合有潜力"
- 包含关键词：方向发现、新方向、研究方向、主题组合、交叉创新、排列组合

## 输入格式
```json
{
  "meeting_ids": ["string (可选，指定从哪些会议中提取主题)"],
  "additional_topics": ["string (可选，手动补充的主题)"],
  "combination_size": "int (可选，组合大小，默认2)",
  "min_novelty_score": "float (可选，最低新颖性阈值，默认0.3)",
  "max_results": "int (可选，最大返回结果数，默认20)",
  "exclude_topics": ["string (可选，排除的主题)"]
}
```

## 执行步骤

### 第一步：提取会议主题
调用 `extract_meeting_topics(meetings)` 从指定的会议记录中提取研究主题关键词：
- 使用 NLP 提取名词短语和技术术语
- 过滤通用词汇，保留专业领域术语
- 合并同义术语（如 "LLM" 和 "大语言模型"）
- 补充用户手动输入的额外主题

### 第二步：生成主题组合
调用 `generate_combinations(topics, k)` 生成所有 k-组合：
- 默认 k=2（两两组合）
- 对于 N 个主题，将产生 C(N, k) 种组合
- 若组合数过多（>200），先按主题相关性预筛选

### 第三步：评估每种组合
对每种组合调用 `evaluate_combination(combo)`，从以下维度评估：

1. **新颖性评分 (novelty_score)**
   - 在 arXiv 搜索该组合的交叉论文数量
   - 论文越少，新颖性越高
   - 完全没有论文可能意味着方向不可行而非创新

2. **可行性评分 (feasibility_score)**
   - 本团队是否有相关能力储备
   - 是否有可获取的数据集和基础设施
   - 技术路线是否可行

3. **趋势对齐度 (trend_alignment)**
   - 组合中的主题是否在全球趋势上升
   - 使用 arXiv 近两年的论文量变化衡量

4. **综合得分 (overall_score)**
   - 加权综合：新颖性×0.35 + 可行性×0.35 + 趋势×0.30

### 第四步：匹配全球趋势
将评估结果与全球研究趋势对照：
- 标注哪些组合恰好处于全球热点
- 标注哪些组合是独特的蓝海方向
- 标注哪些组合与正在衰退的方向有关

### 第五步：生成排名报告
按综合得分排序，为每种有价值的组合生成：
- 中文详细分析（为什么这个方向有潜力）
- 2-3个具体的研究问题建议
- 相关参考论文链接
- 所需的能力和资源评估

## 注意事项

### 创新与可行性平衡
- 不应只推荐最"新奇"的组合，可行性同样重要
- 对于高新颖但低可行性的方向，应说明需要什么条件才能可行
- 对于高可行但低新颖的方向，应说明如何找到差异化切入点

### 避免伪创新
- "把两个热词拼在一起"不等于好的研究方向
- 每个推荐都需要有逻辑上的合理性论述
- 如果某个组合找不到合理的研究动机，不应推荐

### 语言风格
- 使用中文撰写分析报告
- 技术术语保留英文（首次出现时附中文翻译）
- 论述要有深度，不能停留在表面

### 数据时效性
- 确保趋势数据基于最近2年的论文
- 标注数据来源和采集时间
- 承认趋势预测的不确定性

## 关联技能
- `meeting_record`：提供会议记录作为主题提取的数据源
- `team_survey`：全球团队的研究布局可辅助方向评估
- `research_trend`：提供最新的领域趋势数据
- `literature_review`：对有价值的方向可进一步进行文献综述
