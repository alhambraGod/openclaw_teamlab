# 技能：全球团队调研

## 技能名称
`team_survey` — 全球同类科研团队调研与对标分析

## 核心目标
帮助导师了解全球范围内同领域科研团队的研究动态、发表水平和核心方向，通过对标分析识别本团队的相对优势与短板，为团队战略定位和方向调整提供数据支撑。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "帮我看看国内外做类似方向的团队有哪些"
- "和XXX实验室对比一下我们的发表情况"
- "我们团队在这个领域处于什么水平"
- "分析一下竞争团队的研究布局"
- 包含关键词：团队调研、竞争分析、同行对比、团队对标、全球团队

## 输入格式
```json
{
  "research_areas": ["string (必填，团队的核心研究方向)"],
  "known_teams": ["string (可选，已知的对标团队名称)"],
  "time_range_years": "int (可选，分析的时间范围，默认3年)",
  "metrics": ["string (可选，关注的指标: publications, citations, h_index, collaborations)"],
  "focus": "string (可选，'overview'=全景, 'deep_dive'=深入分析某团队)"
}
```

## 输出格式
```json
{
  "similar_teams": [
    {
      "team_name": "XXX Lab",
      "institution": "MIT",
      "leader": "Prof. XXX",
      "homepage": "https://...",
      "research_focus": ["topic1", "topic2"],
      "recent_papers_count": 45,
      "avg_citations": 23.5,
      "top_papers": [{"title": "...", "venue": "NeurIPS 2025", "citations": 120}],
      "relevance_score": 0.85
    }
  ],
  "comparison": {
    "metrics_table": "结构化对比数据",
    "our_strengths": ["在XX方向发表量领先"],
    "our_gaps": ["在XX方向缺少高引论文"],
    "opportunities": ["XX方向是蓝海，竞争较少"]
  },
  "strategic_suggestions": [
    "建议加强与XXX团队在YY方向的合作",
    "建议关注ZZ新兴方向，目前竞争者较少"
  ]
}
```

## 执行步骤

### 第一步：确定调研范围
根据输入的研究方向关键词，确定搜索范围和策略：
- 将中文方向名翻译为英文关键词
- 扩展同义词和相关术语
- 确定搜索的学术数据库和平台

### 第二步：搜索同类团队
调用 `search_similar_teams(research_areas)` 从以下数据源搜索：
- **arXiv**：通过论文作者聚类找到活跃团队
- **Semantic Scholar**：通过引用网络找到关联团队
- **Google Scholar**：通过作者Profile获取团队信息
- **DBLP**：补充计算机科学领域的发表数据

### 第三步：采集团队指标
对每个识别出的团队采集以下指标：
- 近3年发表论文数量及分布（顶会/顶刊比例）
- 论文平均引用量和高引论文
- 团队规模和成员构成
- 研究方向分布和演变
- 合作网络（与哪些团队频繁合作）

### 第四步：对标分析
调用 `compare_metrics(our_team, other_teams)` 进行多维度对比：
- 发表数量对比（折线图数据）
- 影响力对比（引用量、h-index）
- 方向覆盖度对比（韦恩图数据）
- 趋势对比（各团队的方向迁移路径）

### 第五步：生成战略建议
基于对标分析，提出具体的战略建议：
- 需要加强的方向（竞争对手强但我们弱）
- 值得保持的优势（我们强且有护城河）
- 潜在的蓝海方向（全球关注度上升但竞争者少）
- 可能的合作对象（互补性强的团队）

## 注意事项

### 数据准确性
- 学术数据库的数据可能不完整，对比结论需标注数据来源和局限
- 团队边界的界定可能模糊（实验室 vs 课题组 vs 合作网络）
- 引用量等指标受领域差异影响，跨领域对比需谨慎

### 客观性
- 避免过度美化本团队或贬低竞争对手
- 指出差距时同时分析原因（如团队规模、成立时间差异）
- 战略建议基于数据而非主观判断

### 更新频率
- 建议每季度进行一次全面调研
- 重大学术会议（如CVPR、NeurIPS）发布后及时更新
- 保存历史调研结果用于追踪竞争格局变化

### 语言要求
- 报告主体使用中文
- 论文标题、团队名称、机构名称保留英文原名
- 提供关键术语的中英文对照

## 关联技能
- `research_trend`：提供最新论文数据辅助调研
- `direction_discovery`：调研结果可指导新方向发现
- `literature_review`：深入分析某团队时可调用文献综述
