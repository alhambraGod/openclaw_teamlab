# 技能：科研趋势监控

## 技能名称
`research_trend` — 科研趋势每日监控与推送

## 核心目标
每日自动扫描 arXiv、Semantic Scholar 等学术平台，监控团队相关研究领域的最新论文和技术动态，评估每篇论文与团队研究方向的相关度，并生成结构化的趋势报告。确保团队始终了解领域前沿，不错过重要的学术进展。此技能主要由调度器定时触发，实现系统的"自我进化"能力。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "最近有什么新论文"
- "看看今天arXiv上有什么相关的"
- "最近一周的领域动态"
- "监控一下XX方向的新进展"
- 定时触发：调度器每日 6:00 自动执行
- 包含关键词：科研趋势、最新论文、领域动态、前沿追踪、arXiv、热点、emerging

## 输入格式
```json
{
  "domains": ["string (可选，指定监控的领域，默认使用 pi_config 表中的 tracked_domains)"],
  "days": "int (可选，回溯天数，默认1天)",
  "min_relevance": "float (可选，最低相关性阈值，默认0.5)",
  "max_papers": "int (可选，最大返回论文数，默认30)",
  "include_abstract": "bool (可选，是否包含摘要翻译，默认true)"
}
```

## 输出格式
```json
{
  "scan_date": "2026-03-13",
  "domains_scanned": ["cs.AI", "cs.CL", "cs.CV"],
  "total_new_papers": 156,
  "relevant_papers": [
    {
      "title": "论文标题",
      "authors": ["作者1", "作者2"],
      "arxiv_id": "2603.12345",
      "url": "https://arxiv.org/abs/2603.12345",
      "abstract": "原文摘要",
      "abstract_zh": "中文摘要翻译",
      "categories": ["cs.AI", "cs.CL"],
      "relevance_score": 0.85,
      "relevance_reason": "与团队的XX方向高度相关",

  ],
  "trend_summary": {

    "notable_teams": ["值得关注的团队动态"]
  }

```

## 执行步骤

### 第一步：获取跟踪的研究领域
从 `pi_config` 表读取 `tracked_domains` 配置。若用户指定了额外的领域，合并到监控范围中。

### 第二步：扫描学术数据库
调用 `scan_arxiv(domains, days)` 从 arXiv API 获取指定时间范围内的新论文：
- 查询 arXiv API，获取过去 N 天的新论文
- 按研究领域分类
- 提取标题、摘要、作者信息
- 按分类批量查询并去重（同一论文可能出现在多个分类中）

### 第三步：相关性评分
调用 `score_relevance(papers, team_directions)` 评估每篇论文的相关度：
- 将每篇论文与团队的活跃研究方向进行语义匹配
- 计算与每个学生课题的相关度
- 考虑作者是否为已关注的研究者
- 生成相关性得分和中文解释
- 筛选出高相关度论文（relevance > 0.5）

### 第四步：存储趋势数据
将高相关度的论文信息存入 `research_trends` 表，标记为未通知。已经通知过的趋势不重复存储。

### 第五步：趋势分析
基于近期数据进行趋势分析：
- 统计各主题的论文数量变化
- 识别新兴热点主题
- 标注与团队研究方向特别相关的趋势
- 对比本周与上周的论文分布变化

### 第六步：生成报告
将结果整理为结构化报告：
- 按相关性排序的论文列表
- 每篇论文附中文摘要翻译
- 趋势总结和分析

## 注意事项

### 效率与资源
- arXiv API 有速率限制（每3秒一次请求），需合理控制请求间隔
- 大量论文的相关性评分可以异步并行处理
- 缓存已处理过的论文ID，避免重复处理

### 准确性
- 相关性评分不是绝对的，高分论文也可能不相关，低分论文可能有价值
- 建议设置合理的阈值，宁可多推不漏推
- 语义匹配需要考虑中英文对照

### 定时任务
- 每日扫描建议在早晨6:00执行（凌晨arXiv更新后）
- 扫描结果缓存到数据库，避免重复查询
- 异常时（如API不可用）自动重试，并在邮件摘要中说明

### 语言处理
- 论文标题和摘要为英文，需提供准确的中文翻译
- 专业术语翻译要标准（如 Transformer 保留英文，不翻译为"变换器"）
- 翻译质量直接影响导师的阅读效率

## 关联技能
- `email_digest`：扫描结果是邮件摘要的核心内容
- `direction_discovery`：趋势数据辅助方向评估
- `team_survey`：同类团队的新论文是对标分析的输入
- `literature_review`：可进一步对高相关论文进行深度分析
