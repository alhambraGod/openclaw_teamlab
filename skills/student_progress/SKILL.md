# 技能：学生科研进展追踪

## 技能名称
`student_progress` — 学生科研进展追踪与可视化

## 核心目标
帮助导师全面了解每位学生的科研能力成长情况，通过雷达图、趋势分析和里程碑记录，提供直观的学生发展画像。本技能是"个性化指导"和"协作推荐"等下游技能的数据基础。

## 触发条件
当用户（导师）发出以下类型的请求时激活本技能：
- "查看某某同学的科研进展"
- "生成学生能力雷达图"
- "最近几个月的成长趋势如何"
- "某某同学有哪些里程碑事件"
- "对比上次和这次的能力评估"
- 包含关键词：学生进展、能力评估、成长趋势、雷达图、里程碑

## 输入格式
```json
{
  "student_id": "string (必填，学生ID)",
  "student_name": "string (可选，用于模糊查找)",
  "time_range_months": "int (可选，默认6个月)",
  "dimensions": ["string (可选，指定查看的能力维度)"],
  "compare_period": "bool (可选，是否与上一评估周期对比)"
}
```

## 输出格式
```json
{
  "student_profile": {
    "id": "string",
    "name": "string",
    "grade": "string",
    "research_area": "string",
    "advisor": "string"
  },
  "radar_chart_data": {
    "dimensions": ["论文阅读", "实验设计", "代码实现", "学术写作", "团队协作", "创新思维", "汇报表达"],
    "current_scores": [85, 72, 90, 68, 80, 75, 70],
    "previous_scores": [70, 65, 82, 60, 75, 68, 62]
  },
  "growth_trend": [
    {"month": "2025-10", "avg_score": 68.5},
    {"month": "2025-11", "avg_score": 71.2}
  ],
  "milestones": [
    {"date": "2026-01-15", "event": "完成第一篇论文初稿", "dimension": "学术写作"}
  ],
  "summary": "自然语言总结文本"
}
```

## 执行步骤

### 第一步：获取学生基本信息
调用 `fetch_student_profile(student_id)` 从 MySQL 数据库获取学生的基本档案、所属方向、导师信息等。

### 第二步：获取能力雷达图数据
调用 `fetch_radar_data(student_id)` 获取学生在各个能力维度上的最新评分，同时拉取上一评估周期的评分用于对比。

七个核心能力维度：
1. **论文阅读** — 文献检索、精读、批判性分析能力
2. **实验设计** — 实验方案设计、变量控制、数据采集能力
3. **代码实现** — 编程开发、工程实现、工具使用能力
4. **学术写作** — 论文撰写、逻辑表达、学术规范能力
5. **团队协作** — 团队沟通、任务分工、协同完成能力
6. **创新思维** — 提出新想法、跨领域联想、问题重构能力
7. **汇报表达** — 口头汇报、PPT制作、学术答辩能力

### 第三步：生成成长趋势
调用 `fetch_growth_trend(student_id, months)` 获取指定时间范围内各维度得分的变化轨迹，计算月度平均得分趋势。

### 第四步：汇总里程碑事件
从事件记录中筛选该学生的重要里程碑，如：
- 完成论文初稿/终稿
- 通过开题/中期/答辩
- 发表论文/专利
- 获得奖项/荣誉
- 参与重要学术会议

### 第五步：生成自然语言总结
基于以上数据，用中文撰写一段简洁、鼓励性的总结，包含：
- 当前整体水平评价
- 最突出的优势领域
- 最需要关注的成长空间
- 与上一周期的显著变化
- 一到两条具体可行的建议

## 注意事项

### 数据安全
- 学生的评分数据属于敏感信息，仅向该学生的导师展示完整数据
- 学生本人查看时，侧重展示成长趋势和鼓励，弱化横向对比

### 语言风格
- 使用中文回复，语气温和、专业
- 总结部分应当鼓励为主，建设性批评为辅
- 避免使用过于绝对的评价词汇（如"很差"、"不行"）
- 用"有提升空间"代替"薄弱"，用"值得关注"代替"落后"

### 数据完整性
- 若某维度缺少评分数据，标注为"暂无数据"而非0分
- 若学生为新入学，说明数据积累不足，建议一个月后再次评估
- 确保时间范围内的趋势数据点足够（至少3个月）才生成趋势分析

### 可视化建议
- 雷达图使用七边形，各维度均匀分布
- 当前评分用实线填充，上一周期用虚线叠加
- 趋势图使用折线图，标注关键事件节点
