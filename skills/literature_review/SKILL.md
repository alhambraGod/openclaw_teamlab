# 技能：系统性文献综述

## 技能名称
`literature_review` — 系统性文献检索与综述撰写

## 核心目标
帮助学生和导师高效完成文献综述工作，包括文献检索、筛选、分类、分析和综述文本撰写。将原本需要数周的文献调研压缩到数小时，同时保证文献覆盖的全面性和分析的深度。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "帮我做一个XX方向的文献综述"
- "搜索一下XX领域最近的论文"
- "整理一下XX话题的相关工作"
- "帮我写 Related Work 部分"
- 包含关键词：文献综述、文献检索、文献分析、相关工作、Related Work、文献调研

## 输入格式
```json
{
  "topic": "string (必填，综述主题)",
  "keywords": ["string (可选，补充搜索关键词)"],
  "time_range_years": "int (可选，检索时间范围，默认5年)",
  "max_papers": "int (可选，最大论文数，默认50)",
  "venues": ["string (可选，指定会议/期刊，如 'NeurIPS', 'ICML')"],
  "output_format": "string (可选，'summary'=摘要列表, 'review'=综述文本, 'both')",
  "language": "string (可选，'zh'=中文, 'en'=英文, 默认'zh')"
}
```

## 输出格式
```json
{
  "search_stats": {
    "total_found": 320,
    "filtered": 50,
    "time_range": "2021-2026",
    "sources": ["arXiv", "Semantic Scholar", "DBLP"]
  },
  "papers_by_category": {
    "方法类": [{"title": "...", "authors": "...", "year": 2025, "summary": "...", "relevance": 0.9}],
    "应用类": [],
    "理论类": []
  },
  "review_text": "综述正文（Markdown格式）",

```

## 执行步骤

### 第一步：关键词扩展与检索策略制定
- 将用户输入的主题扩展为多组搜索关键词
- 考虑同义词、缩写和相关术语
- 制定分层检索策略：核心关键词 → 扩展关键词 → 交叉关键词

### 第二步：多源文献检索
从以下数据源并行检索：
- arXiv（预印本，最新成果）
- Semantic Scholar（综合性，含引用数据）
- DBLP（计算机科学文献索引）
- Google Scholar（最广泛的覆盖范围）

### 第三步：文献筛选与去重
- 按标题和 DOI 去重
- 按相关性评分筛选（基于标题和摘要与主题的语义相似度）
- 按影响力筛选（引用量、发表venue等级）
- 保留 top-N 篇最相关论文

### 第四步：文献分类与分析
将筛选后的文献按以下维度分类：
- **方法论分类**：提出了什么新方法/模型/算法
- **应用场景分类**：应用在什么领域/任务
- **时间线梳理**：技术演进的时间脉络
- **引用关系**：关键论文之间的引用链

### 第五步：综述文本生成
生成结构化的综述文本，包括：
- 引言（领域背景和研究动机）
- 分类综述（按主题分节讨论）
- 方法对比（表格形式对比主要方法）
- 研究空白与未来方向
- 参考文献列表

## 注意事项

### 学术诚信
- 明确标注每篇论文的来源和引用信息
- 综述文本必须是原创的分析和总结，不能直接复制论文内容
- 区分"基于AI辅助的综述"和"完全AI生成的综述"

### 覆盖全面性
- 不能只关注高引论文，也要包含有创新性但引用较少的新论文
- 确保正反观点都被涵盖
- 注意标注检索可能遗漏的领域

### 语言质量
- 综述文本的学术语言规范
- 中文综述中，专业术语首次出现时附英文原文
- 参考文献格式统一（建议使用 APA 或 IEEE 格式）

### 时效性
- 标注数据检索日期
- 对于快速发展的领域，提醒用户定期更新综述

## 关联技能

- `direction_discovery`：文献综述可为方向发现提供证据
- `academic_writing`：综述文本可直接嵌入论文的 Related Work 部分
