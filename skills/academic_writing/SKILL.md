# 技能：学术写作辅助

## 技能名称
`academic_writing` — 学术论文写作辅助

## 核心目标
为团队成员提供全方位的学术论文写作支持，包括论文结构设计、段落撰写与润色、逻辑梳理、语言优化和格式规范。帮助学生快速提升学术写作能力，同时保证论文质量符合目标会议/期刊的要求。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "帮我润色一下这段论文"
- "这段Introduction怎么写比较好"
- "帮我改改这篇论文的逻辑结构"
- "这段英文表达有什么问题"
- "帮我写一下Abstract"
- 包含关键词：学术写作、论文润色、写作辅助、论文修改、语言润色、逻辑梳理

## 输入格式
```json
{
  "text": "string (必填，需要处理的文本)",
  "task_type": "string (必填，任务类型: 'polish'=润色, 'structure'=结构, 'rewrite'=重写, 'review'=审阅)",
  "section": "string (可选，论文部分: 'abstract', 'introduction', 'method', 'experiment', 'conclusion')",
  "target_venue": "string (可选，目标会议/期刊名称)",
  "language": "string (可选，'en'=英文, 'zh'=中文, 默认'en')",
  "style_notes": "string (可选，额外的风格要求)"
}
```

## 输出格式
```json
{
  "revised_text": "string (修改后的文本)",
  "changes": [
    {
      "original": "原文片段",
      "revised": "修改后片段",
      "reason": "修改理由",
      "category": "grammar|logic|style|clarity|conciseness"
    }
  ],
  "overall_feedback": "整体评价和建议",
  "writing_score": {
    "clarity": 8,
    "logic": 7,
    "grammar": 9,
    "academic_tone": 8,
    "overall": 8
  },
  "improvement_tips": ["具体的写作改进建议"]
}
```

## 执行步骤

### 第一步：理解写作任务
分析用户的写作需求：
- 确定是润色、重构还是从头撰写
- 了解目标论文的部分（Introduction / Method / Experiment 等）
- 确定目标语言和学术风格
- 如有目标venue，了解其写作风格偏好

### 第二步：文本分析
对输入文本进行多维度分析：
- **语法检查**：时态一致性、主谓一致、冠词使用等
- **逻辑检查**：段落间逻辑连贯性、论证完整性
- **风格检查**：学术语气、表达简洁性、被动/主动语态
- **术语检查**：专业术语使用的一致性和准确性

### 第三步：修改与润色
根据分析结果进行修改：
- 逐句检查并优化语言表达
- 确保段落内的逻辑链清晰
- 调整句式长度和多样性
- 消除冗余表达

### 第四步：生成修改说明
为每处修改提供详细说明：
- 标注原文和修改后的对比
- 解释修改理由（不仅改，还要教）
- 按修改类型分类（语法/逻辑/风格/简洁性）

### 第五步：输出整体反馈
提供宏观层面的写作建议：
- 文本整体质量评分
- 最突出的优点
- 最需要改进的方面
- 3-5条具体可行的写作改进建议

## 注意事项

### 教育导向
- 写作辅助的目标是帮助学生"学会写"，而非"代替写"
- 每处修改都要附带理由，帮助学生理解为什么要这样改
- 鼓励学生先独立完成初稿，再使用本技能进行润色

### 学术规范
- 不编造数据或实验结果
- 不改变作者的原始观点和实验结论
- 保持论文的原创性和学术诚信

### 针对中国学生的常见问题
- 中式英语表达（Chinglish）的纠正
- 长句拆分（中文思维倾向于写长句）
- 逻辑连接词的恰当使用
- 被动语态和主动语态的平衡
- 冠词（a/an/the）的准确使用

### 语言风格
- 修改说明使用中文（方便学生理解）
- 修改后的论文文本保持目标语言
- 语气鼓励、建设性

## 关联技能
- `literature_review`：综述文本可直接进入写作润色流程
- `student_progress`：写作能力评估可反馈到学生的"学术写作"维度得分
- `meeting_record`：从组会汇报中获取写作改进需求
