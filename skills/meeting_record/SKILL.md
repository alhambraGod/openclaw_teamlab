# 技能：会议记录与总结

## 技能名称
`meeting_record` — 会议记录整理与智能总结

## 核心目标
将会议的原始笔记或语音转录文本整理为结构化的会议纪要，自动提取行动项（Action Items），追踪任务分配和完成情况。减少导师和学生在会议整理上的时间消耗，确保会议讨论成果被完整记录和追踪。

## 触发条件
当用户发出以下类型的请求时激活本技能：
- "帮我整理一下今天的会议记录"
- "总结一下刚才的讨论"
- "提取一下这次会议的行动项"
- "上次组会布置的任务完成情况怎样"
- 包含关键词：会议记录、会议纪要、会议总结、行动项、组会记录

## 输入格式
```json
{
  "raw_notes": "string (必填，原始会议笔记或转录文本)",
  "meeting_type": "string (可选，'group_meeting'=组会, 'one_on_one'=一对一, 'brainstorm'=头脑风暴)",
  "participants": ["string (可选，参会人员列表)"],
  "date": "string (可选，会议日期，默认今天)",
  "previous_action_items": ["string (可选，上次会议的行动项，用于追踪)"]
}
```

## 执行步骤

### 第一步：文本预处理
对原始笔记进行预处理：
- 识别发言人（如有标注）
- 分段落和分主题
- 纠正明显的语音识别错误（如为转录文本）
- 处理中英文混合的情况

### 第二步：生成会议摘要
调用 `summarize_notes(raw_notes)` 生成结构化摘要：
- 提取会议主题和关键议题
- 为每个议题总结讨论要点
- 记录达成的共识和决策
- 标注未解决的开放问题

### 第三步：提取行动项
调用 `extract_action_items(notes)` 智能识别行动项：
- 识别包含任务分配的句子（如"XX负责..."、"下周完成..."）
- 提取任务描述、负责人、截止日期
- 为缺失信息标注待确认
- 按优先级排序

### 第四步：追踪上次任务
如果提供了上次会议的行动项：
- 在本次会议内容中搜索任务完成的相关讨论
- 自动标注任务状态（已完成/进行中/逾期）
- 生成任务追踪报告

### 第五步：提取研究主题
从会议讨论中提取有价值的研究主题关键词：
- 用于 `direction_discovery` 技能的主题输入
- 标注讨论的热度（被多少人提及/讨论时间长度）

### 第六步：持久化存储
将会议记录保存到数据库：
- 会议基本信息和摘要
- 结构化的讨论要点
- 行动项（关联到任务追踪系统）
- 提取的主题关键词

## 注意事项

### 准确性
- 会议摘要应忠实于原始讨论内容，不添加未讨论的内容
- 行动项的负责人和截止日期必须准确
- 如果原始笔记中信息不明确，标注"待确认"而非猜测

### 保密性
- 一对一会议的内容仅对参会者可见
- 涉及学生个人问题的讨论不在公开摘要中展示
- 敏感决策标注"仅限内部"

### 语言风格
- 会议纪要使用中文
- 简洁、条理清晰
- 行动项使用动词开头（如"完成..."、"准备..."、"提交..."）
- 避免记录口语化的表达和无关闲聊

### 与其他技能的联动
- 提取的主题自动输入到 `direction_discovery`
- 学生的表现观察可输入到 `student_progress` 更新能力评分
- 讨论中产生的写作需求可转交 `academic_writing`

## 关联技能
- `direction_discovery`：提供主题关键词
- `student_progress`：更新学生表现记录
- `academic_writing`：处理会议中产生的写作需求
- `email_digest`：会议摘要可纳入每日推送
